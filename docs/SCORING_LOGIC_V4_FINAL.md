# AlphaPick V4 종합 점수 및 매매 신호

## 목적

V4는 좋은 기업과 좋은 매수 시점을 혼동하지 않는다. 모든 종목은 세 축을 독립적으로 계산한 뒤 종합 투자매력도를 산출한다.

- **회사 품질(Q)**: 중장기 보유 가치
- **시장 검증(M)**: 가격이 실제 시장에서 상대적으로 검증됐는지
- **매수 타이밍(T)**: 지금 진입이 적절한지

뉴스와 공시는 판단 근거로 제공하지만 점수에는 직접 반영하지 않는다.

## 종합 점수

```text
Composite = 100 × (max(Q, 5) / 100)^0.40
                  × (max(M, 5) / 100)^0.25
                  × (max(T, 5) / 100)^0.35
             + Valuation adjustment
```

`max(..., 5)`는 기하평균이 0으로 붕괴하는 것을 막기 위한 종합 계산 전용 하한이다. 개별 점수와 매매 신호의 실제 값은 그대로 유지한다. Q, M, T 가운데 하나라도 계산할 수 없으면 종합 점수는 순위에서 제외한다.

밸류에이션 조정은 비교 가능한 PER/PBR이 있을 때만 적용한다.

| 상태 | 조정 |
| --- | ---: |
| 저평가 | +4 |
| 적정가 | 0 |
| 고평가 | -3 |
| 과대평가 | -7 |
| 비교 불가 | 0 |

## 회사 품질 Q (100점)

| 항목 | 비중 | 현재 데이터 기준 |
| --- | ---: | --- |
| 실적 성장성 | 30 | EPS 성장률, 이후 OpenDART 분기 매출·영업이익 YoY로 보강 |
| 수익성·자본 효율 | 30 | ROE, 영업이익률 |
| 재무 안정성 | 25 | 부채비율, 이후 현금·이자보상배율 보강 |
| 현금흐름·이익의 질 | 15 | OpenDART 현금흐름 수집 전에는 중립값과 `partial` 상태 표시 |

최근 2분기 연속 영업적자, 자본잠식, 거래정지 등 치명적 결함은 투자 부적격으로 표시하고 종합 순위에서 제외한다.

## 시장 검증 M (100점)

| 항목 | 비중 |
| --- | ---: |
| 12-1개월 상대강도 | 40 |
| 6-1개월 상대강도 | 20 |
| 하방 변동성 방어 | 25 |
| 최대낙폭(MDD) 방어 | 15 |

상대강도는 같은 계산일 유니버스 내 백분위로 계산해 시장 수준이 달라도 비교 가능하게 한다.

## 매수 타이밍 T (100점)

| 항목 | 비중 | 주요 판정 |
| --- | ---: | --- |
| 추세 | 30 | EMA20/EMA50 상단, EMA 정배열 |
| 수급 | 25 | 거래량 비율, OBV 방향 |
| 돌파 품질 | 25 | 52주 고점 근접도, 거래량 급증 |
| 진입 품질 | 20 | 단기 Z-Score 과열 여부 |

Z-Score가 2.5를 넘으면 타이밍에 강한 할인율을 적용하며, 전체 시장이 약세면 타이밍을 추가로 할인한다. 이것은 좋은 회사를 무리하게 추격 매수하지 않도록 하기 위한 규칙이다.

## 매매 신호 워터폴

아래 규칙은 위에서부터 먼저 적용한다.

1. 거래정지·치명적 결함·평가 불가: `평가 보류`
2. EMA50 하향 이탈과 OBV 악화: `신규 진입 회피`
3. 과열과 추세 훼손 동시 발생: `분할매도 검토`
4. 단기 과열: `추격 매수 대기`
5. 회사 품질 약함: `단기 트레이딩 검토`
6. 시장 검증 약함: `추세 확인 대기`
7. 타이밍 70점 이상: `분할매수 검토`
8. 타이밍 50점 이상: `관망`
9. 그 외: `진입 보류`

## 운영과 UI

- 일별 배치: 가격 기반 M/T, 종합 점수, 매매 신호를 재계산한다.
- 분기 배치: OpenDART 재무 데이터로 Q를 재계산한다.
- 목록 API는 `sort=composite|company|market|timing|valuation`와 `direction=asc|desc`를 지원한다.
- 목록에서는 종합 점수, Q/M/T, 밸류 조정, 매매 신호를 함께 보여준다.
- 상세 페이지는 각 축의 산식, 원천 지표, 할인 및 경고 사유를 보여주는 것을 원칙으로 한다.
